What are Semantic Elements?

Semantic elements clearly describe their meaning in a human and machine-readable way.


They provide structure and meaning to the content they enclose, improving accessibility and SEO.


Using semantic elements helps create a well-structured and meaningful HTML document.

Header Nav Main Article & Section Aside Footer

<header>

  • used to represent introductory content or a group of navigational links for a section of a document or the entire document
  • improves accessibility and SEO by explicitly indicating to browsers and search engines that the enclosed content is introductory or navigational
  • commonly found at the top of a webpage, but can also be used within other sectioning elements like <article> or <section> to provide a header for that specific section
  • helps organize the document's structure, making the HTML code cleaner and more readable
<header>
  <h1>FrontEnd Insights</h1>
  <h2>Resource for modern Front-End web development</h2>
</header>

FrontEnd Insights

Resource for modern Front-End web development

Best Practices

  • Use for introductory content or navigation
  • Can be used multiple times per page
  • Should not contain <footer> elements

<nav>

  • used to reprsent a section of a page that contains navigation links, either to other pages within the same website or to different sections within the current page
  • explicitly marks a block of navigation links, improving accessibility and SEO by making the page structure clearer
  • a page can have multiple <nav> elements if there are distinct navigation sections
  • can include breadcrumbs for better navigation
<nav>
  <ul>
    <li><a href="#">Home</a></li>
    <li><a href="#">About</a></li>
    <li><a href="#">Services</a></li>
    <li><a href="#">Contact</a></li>
  </ul>
</nav>
Home About Services Contact

Best Practices

  • Use for major navigation blocks
  • Don't overuse - not every link group needs nav
  • Can be used multiple times per page

<main>

  • used to represent the dominant content of the <body> of a document
  • should be unique to the document and not contain content that is repeated across documents such as sidebars, navigation links, or footers
  • improves accessibility by allowing assistive technologies to quickly locate the main content of the page
  • helps search engines understand the primary topic of the page, potentially improving SEO
<main>
  <h1>Welcome to Our Website</h1>
  <p>Here is the main content of the page...</p>
</main>

Best Practices

  • Only one <main> element per page
  • Should contain primary content
  • Should not contain headers, footers, sidebars

<article>

  • used to represent a self-contained content that is, in principle, independently distributable or reusable
  • content should make sense on its own, even if removed from the surrounding context of the webpage
  • <article> elements can be nested, the inner <article> elements represent content that is related to the outer <article> elements

<section>

  • used to represent a generic standalone section of a document or application
  • used to group related content that forms a thematic unit, often with a heading
  • use <section> when there isn't a more specific semantic element available (like <article>, <aside>, or <nav>) to represent the content

While both are sectioning elements, <article> is for standalone, independently distributable content, whereas <section> groups related content within a larger document or another section.

<article>
  <h2>News Story 1</h2>
  <p>Here is the story content...</p>
</article>

<article>
  <h2>News Story 2</h2>
  <p>Here is the story content...</p>
</article>
<section>
  <h2>Our Products</h2>
  <p>this is the section about our products...</p>
</section>

<section>
  <h2>Our Team</h2>
  <p>this is the section about our team...</p>
</section>

Best Practices

  • Use <article> for content that can stand alone (e.g., blog posts, news articles)
  • Use <section> for grouping related content within a page (e.g., chapters, thematic groups)
  • Both can be nested within each other as needed

<aside>

  • used to represent content that is related to the main content but is not essential to its understanding
  • often contains sidebars, pull quotes, advertisements or other tangentially related information
  • can improve accessibility by providing additional context or information
  • helps search engines understand the structure of the page and the relationship between different content
<aside>
  <h4>Related</h4>
  <p>Content...</p>
</aside>
Best Practices:
  • Use <aside> for tangentially related content
  • Keep <aside> content concise and relevant
  • Can be used multiple times per page

Text-Level Semantic Elements

Text-level semantic elements provide meaning to inline content, enhancing the structure and readability of text.

Help convey the importance, emphasis, or role of specific text within a document.

If there is no semantic purpose, do not use these elements for purely presentational reasons.


<strong> : Indicates strong importance, seriousness, or urgency for its contents


<em> : Indicates emphasized text, typically displayed in italics.


<mark> : Highlights text for reference or emphasis.


<small> : Renders text in a smaller font size, often for fine print.


<cite> : Indicates the title of a creative work, like a book or song.

<abbr> : for HTML abbreviations


<code> : for inline code


<kbd> : for Ctrl+C keyboard input


<var> : for variables in code


<s> : Represents text that is no longer accurate or relevant.

<time>

  • used to represent a represent a specific period in time
  • can include dates, times, or durations
  • primary purpose of this element is to provide a machine-readable format of the date or time for applications like screen readers or search engines
  • datetime attribute is crucial for providing a machine-readable format of the date or time
The datetime attribute of the HTML element can represent a duration using the ISO 8601 duration format.

P[nD][T[nH][nM][nS]]

Valid year string: YYYY (e.g., 2025)

<p>
  You must be born before
<time datetime="2003">2003</time> to enter.
</p>

Valid month string: YYYY-MM (e.g., 2025-12)

<p>
  The event ends in
<time datetime="2026-02">February 2026</time>.
</p>

Valid date string: YYYY-MM-DD (e.g., 2025-12-25)

<p>
  The event starts on
<time datetime="2025-12-25">December 25th, 2025</time>.
</p>

Valid time string: HH:MM (e.g., 14:30)

<p>
  The meeting is scheduled for
<time datetime="14:30">2:30 pm</time>.
</p>

Valid time string: HH:MM:SS (e.g., 14:30:00)

<p>
  The meeting is scheduled for
<time datetime="14:30:00">2:30 pm</time>.
</p>

Valid time string: HH:MM:SSZ (e.g., 14:30:00Z for UTC time)

<p>
  The meeting is scheduled for
<time datetime="14:30:00Z">2:30 pm</time>.
</p>

Best Practices

  • Always include the datetime attribute
  • Use a machine-readable format (e.g., YYYY-MM-DD)
  • Provide human-readable text within the element

<address>

  • used to provide contact information for the nearest <body> or <article> ancestor
  • typically relates to the author, owner, or maintainer of the content
  • improves accessibility by clearly indicating contact information to assistive technologies
<address>
  Contact info here...
</address>

Should enclose contact details such as

  • physical address: Street, city, state/province, postal code, country
  • email address: often presented as a mailto: link
  • phone number
  • social media handles or URLs: related to the contact person or organization
  • other relevant contact information

<figure>

  • used to represent self-contained content, such as illustrations, diagrams, or photos
  • typically includes a caption or legend to describe the content
  • can be referenced from the main content without disrupting the flow of the document
  • improves accessibility by providing context for visual content

<figcaption>

  • used to represent the caption or legend for a <figure> element
  • typically includes a description of the content
  • must be the first child of the <figure> element
  • improves accessibility by providing context for visual content
<figure>
  <img src="image.jpg" alt="Description">
  <figcaption>Caption</figcaption>
</figure>
Best Practices:
  • Use <figure> for self-contained content
  • Include a <figcaption> for descriptive text
  • Ensure <figure> elements are referenced in the main content

The <details> and <summary> elements work together to create a native disclosure widget

<details>

  • creates a disclosure (disclosure pattern) widget, allowing you to provide additional information that can be shown or hidden by the user
  • inherently provides expand/collapse functionality, that is closed by default, without requiring JavaScript for basic operation
  • use <details> for content that is not immediately necessary, but may be useful for users who want more information

<summary>

  • provides the always visible heading or title for the collapsible section
  • ensure the <summary> element is always the first child of the <details> element
  • can contain text, images, or other inline elements to describe the content that will be revealed

The content inside <details> is hidden until the user interacts with the <summary>

<details>
  <summary>Click to reveal more details</summary>
  <p>Hidden Content</p>
  <ul>
    <li>Item 1</li>
    <li>Item 2</li>
  </ul>
</details>

<cite>

  • the citation element is used to define the title of a creative work, such as a book, article, song, movie, or website
  • is typically rendered in italics by browsers
<p>
My favorite movie is <cite>Where The Heart Is</cite> staring Natalie Portman.
</p>

My favorite movie is Where The Heart Is staring Natalie Portman.

<q>

  • the inline quotation element is used for short, inline quotations that do not require paragraph breaks
  • typically relates to the author, owner, or maintainer of the content
  • browsers typically render the enclosed text with quotation marks
  • can include a cite attribute, which provides a URL pointing to the source document or message for the quoted content
  • the cite attribute is primarily for machine readability and is not directly displayed to the user
<p>Someone yelled,<q>FIRE</q> and everyone ran.</p>

Someone yelled, FIRE and everyone ran.

<blockquote>

  • the block quotation element is used for longer quotations that require paragraph breaks, displayed as a separate block, often with indentation
  • can include a cite attribute, which provides a URL pointing to the source document or message for the quoted content
  • the cite attribute is primarily for machine readability and is not directly displayed to the user
<blockquote>
 <p>
   Butterflies go through four major stages of metamorphosis:
   egg, caterpillar, chrysalis and adult.
 </p>
 <footer>
  <cite title="The Butterfly Website">
   The Butterfly Website
  </cite>
 </footer>
</blockquote>

"Butterflies go through four major stages of metamorphosis: egg, caterpillar, chrysalis and adult."

The Butterfly Website
